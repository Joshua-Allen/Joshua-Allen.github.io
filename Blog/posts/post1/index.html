Today I want to remake how I handle the environmental objects in the map room. The environmental object handling and creation code is very old so instead of hacking my way around it I’ve decided that it really should just be remade.

Whenever I want to recode some part of the game, the first thing that I do is comment out everything that has to do with that code, if needed I put in some temporary variable, and test the game to see if everything works. I do this because I really don’t want to miss anything.

The only real requirements for this system are:

  1. Tim needs to be able to edit it easily
  2. Have the ability to be expanded without a rewrite (missed this the first time around)


This first requirement might seem easy but when working with game maker I have to do some weird things that I would take for granted if I were to let’s say be working in java. My own little technique for easy editing comes from the way GameMaker scripts work.

///scr_map_add_object(sprite, chance, yMin, yMax, xTimerMin, xTimerMax, yTimerMin, yTimerMax); /* sprite - the sprite that the background object will use chance - the precent chance that the object will be created yMin yMax - the object will created in the range of yMin and yMax xTimerMin xTimerMax - on the creation of the object xtimer will be set with a value in the range of xTimerMin and xTimerMax yTimerMin yTimerMax - on the creation of the object ytimer will be set with a value in the range of yTimerMin and yTimerMax */ for (var i=0; i<argument_count; i++) { obj_info[curIndex,i] = argument[i]; } curIndex++;

This little script populates the second index of the array with the arguments and then increments the index for when the next time the script is called. This technique only works because I know it will be only used within one piece of code in a known location. It is not as fast as just having the code all in one place but one line for each object is much easier to look at then ten.

GameMaker does not allow for a true object constructer so once again I must use a little technique that allows for me to send a lot of data with the creation of the environment objects. This technique requires a known object that is already created and an array or other data stricter within it. Because of GameMaker’s object creation code “instance_create(x, y, obj )“ I can send two doubles to the object before it runs the creation code, so in that bit of cade I can do something like:

// the index of the array to look at info_index = y; // get the info sprite_index = info_array[info_index,0]; y = irandom_range(info_array[info_index,2], info_array[info_index,3]); xTime_max = irandom_range(info_array[info_index,4], info_array[info_index,5]); yTime_max = irandom_range(info_array[info_index,6], info_array[info_index,7]);

So when creating a new environmental object I must remember to set the y to the index of the array. There are better ways of going about this, like having a global data structure for passing variables like this.
